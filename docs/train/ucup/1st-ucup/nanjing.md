## A. Stop, Yesterday Please No More

!!! info "题意"
    - 给定一张 $n \times m$ 的网格，某个格子上有一个洞，其余格子都是有一只袋鼠的空地。
    - 所有袋鼠会同时按照 `UDLR` 操作移动，踩到洞或掉出网格的袋鼠会消失。
    - 给定一个操作序列 $S$，已知执行后还剩 $k$ 只袋鼠，求一开始有多少位置可能是洞。
    - $1 \leq n, m\leq 10^3$，$1 \leq |S| \leq 10^6$。

??? tip "题解"
    - 考虑把移动过程视作：所有袋鼠不动，洞和网格边界一起动。
    - 那么如果不考虑洞，最后剩下的袋鼠会是一个连续子矩形，可以直接求出。
    - 引入洞的影响后，相当于有一条固定的路径形状，问有多少起点能让其与该子矩形的交恰好为某定值。
    - 直接二维差分即可求出每个起点出发后与子矩形的交。时间复杂度 $\mathcal O(nm + |S|)$。

## B. Ropeway

!!! info "题意"
    - 给定序列 $[a_1, a_2, \ldots, a_n]$ 和常数 $k$。有 $q$ 次 **临时** 的单点修改，每次修改后：
        - 选择一个子序列 $[a_{p_1}, a_{p_2}, \ldots, a_{p_m}]$，使得 $p_i - p_{i-1} \leq k$（令 $p_0=0,p_{m+1}=n+1$）。
        - 最小化 $a_{p_i}$ 之和。输出这个最小值。
    - $1 \leq n \leq 5\times 10^5$，$1 \leq k, q \leq 3\times 10^3$。
??? tip "题解"
    - 对于没有修改的情况，可以单调队列优化 dp 做到 $\mathcal O(n)$。
    - 询问时，只需要讨论修改的位置是否被包含在最优方案中，即可得到答案式。计算的复杂度是 $\mathcal O(k)$。
    - 预处理正反 dp，总时间复杂度 $\mathcal O(n + qk)$。

## C. Fabulous Fungus Frenzy

!!! info "题意"
    - 给定两个 $n \times m$ 的矩阵 $A$ 和 $B$，以及 $k$ 个模式矩阵。有三种可执行操作：
        - 交换 $A$ 中的两个相邻位置。
        - 顺时针旋转一个 $A$ 中的 $2 \times 2$ 子矩阵。
        - 选择一个模式矩阵，用其覆盖 $A$ 中的某个子矩阵。
    - 用不超过 $4\times10^5$ 次操作、不超过 $400$ 次第三种操作把 $A$ 变成 $B$，或报告无解。
    - $n, m, k \leq 20$。
??? tip "题解"
    - 考虑倒着做。一二操作不变，三操作变为把一个能匹配模式矩阵的子矩阵变成通配符。
    - 由于一二操作的存在，我们实际上可以任意重排矩阵内的元素。因此判定某个模式矩阵是否可用是容易的。
    - 那么贪心地不断使用可用的模式矩阵，直到全体元素都变成通配符即可（把 $B$ 也视为模式矩阵）。
    - 注意到每个模式矩阵第一次可用后，就可以一个一个把所有其含有的字符全变成通配符。利用这一点进行一些精细分析，可知直接这样贪心是满足操作次数限制的，时间复杂度也可以接受。

## D. Chat Program

!!!info "题意"
    - 给定序列 $[a_1, a_2, \ldots, a_n]$ 和 $k, m, c, d$。可以进行如下操作至多一次：
        - 选择一个长为 $m$ 的区间，把里面的第 $i$ 个数加 $c + d(i-1)$。
    - 最大化序列第 $k$ 大的值。
    - $n \leq 2\times 10^5$。

??? tip "题解"
    - 二分答案 $v$。相当于求能否让 $\geq v$ 的数有至少 $k$ 个。
    - 显然每个元素有贡献的情况对应的首项位置是个区间。算出两端点再差分前缀和即可。
    - 时间复杂度 $\mathcal O(n \log V)$。

## E. Color the Tree

!!!info "题意"
    - 给定一棵 $n$ 个点的有根树，以及序列 $[a_0, a_1, \ldots, a_{n-1}]$。
    - 可以以 $a_i$ 的代价，选择任意节点 $u$，把所有 $u$ 子树内距其为 $i$ 的点染成黑色。
    - 一开始所有点都是白色的，求把所有点染黑的最小代价。
    - $n \leq 10^5$。
